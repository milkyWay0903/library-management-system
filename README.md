# Java实验三 - 图书馆管理系统
一、实验目的
1.理解 Java 集合框架核心组件（List、Set、Map）的基本使用
2.掌握泛型类、泛型方法的基础应用
3.学会在实际场景中选择合适的集合类型
4.理解集合的排序、查找和遍历操作
5.掌握泛型带来的类型安全优势
二、实验环境
oJDK 11 或更高版本
oIntelliJ IDEA 或 Eclipse 等 Java 开发环境
oWindows/macOS操作系统
三、实验内容
任务 1：设计图书和用户等核心实体类
任务 2：创建一个简单的泛型工具类，展示泛型方法的应用
任务 3：使用集合框架实现图书馆的核心功能
任务 4：创建简单的控制台交互程序
四、实验说明
核心集合应用
1.Map 集合
booksByIsbn: 使用HashMap存储 ISBN 与图书的映射，实现快速查找
booksByType: 使用EnumMap按图书类型分组，适合枚举键的场景
usersById: 使用HashMap存储用户 ID 与用户信息的映射
2.Set 集合
存储图书标签，利用HashSet的特性确保标签不重复
按类型存储图书，确保同类型图书不重复存储
3.List 集合
borrowRecords: 使用ArrayList存储借阅记录，适合频繁添加和遍历
预约列表：使用ArrayList管理预约队列，支持按顺序通知
泛型应用
1.泛型类
集合框架本身大量使用泛型（List<Book>、Map<String, User>）
确保集合只能存储指定类型的对象，提供编译时类型安全
2.泛型方法
CollectionUtils中的findFirst()和filter()方法
实现了与具体类型无关的通用功能，提高代码复用性
五、实验步骤
1.创建所有类并确保代码编译通过
2.运行LibrarySystem主类，体验图书馆系统功能
3.测试不同的集合操作：
添加图书和用户
搜索和借阅图书
归还图书
4.分析代码中集合类型的选择
5.尝试修改代码，使用不同的集合实现类（如TreeMap替代HashMap），观察效果差异
六、实验拓展
1.为图书添加评分功能，使用Map<Book, Double>存储评分
2.实现图书分类统计功能，使用Map<BookType, Integer>记录各类图书数量
3.添加简单的图书排序功能，按出版日期或作者排序
4.扩展CollectionUtils，添加泛型的集合复制方法
